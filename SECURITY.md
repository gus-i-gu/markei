# Security Policy

## Reporting a vulnerability

Please do not disclose suspected security or privacy vulnerabilities in public issues, discussions, screenshots, or sample databases.

Report the matter privately to the repository owner through an available private GitHub contact channel. Include:

- the affected commit, version, or build;
- the environment in which it was observed;
- concise reproduction steps;
- the potential impact;
- logs or evidence with all personal data removed;
- whether the issue appears to affect local storage, synchronization, authentication, packaging, or contributed analytics.

Do not access, copy, retain, or publish another person's household data while investigating a problem.

## Supported versions

Markei is currently pre-release software. Security review focuses on the active development line and the most recent distributed beta. Older experimental branches and builds may not receive fixes.

## Disclosure

Please allow a reasonable opportunity to investigate and prepare a correction before public disclosure. No bounty program or guaranteed response-time agreement currently exists.
